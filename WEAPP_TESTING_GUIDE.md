# 轻记（测试阶段）部署与联调教程

## 1. 启动后端（本地）

进入 [backend/README.md](file:///e:/PersonalFiles/Coding/payBook/backend/README.md) 按步骤安装依赖并启动：

```bash
cd backend
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

默认接口前缀：`http://localhost:8000/api/v1`

## 2. 小程序端配置后端地址

打开小程序 → Tab「我的」→ 修改“后端地址”为：

- 本地联调：`http://localhost:8000/api/v1`
- 远程联调：`https://你的域名/api/v1`

保存后切回首页会自动刷新。

注意：如果你是在云端预览页面/远程预览环境中运行（非本机微信开发者工具），`localhost` 指向的是远程机器，不是你的电脑，因此会请求失败。此时需要把后端部署到公网 HTTPS（或内网穿透）再填入。

补充：如果当前页面是 HTTPS（例如云端预览链接），浏览器会阻止请求 `http://...`（Mixed Content），也会表现为 `Failed to fetch`。解决方式同上：使用公网 HTTPS 后端地址。

## 3. 微信开发者工具联调要点（测试阶段）

### 3.1 合法域名限制

测试阶段推荐在微信开发者工具中开启：

- 详情 → 本地设置 → 勾选 “不校验合法域名、web-view(业务域名)、TLS 版本以及 HTTPS 证书”

这样可以直接请求 `http://localhost:8000`（仅限开发者工具/调试，真机与线上必须走 HTTPS 合法域名）。

### 3.2 真机联调（需要 HTTPS）

真机调试时，必须让后端通过 HTTPS 域名可访问：

- 购买/使用已有备案域名 + Nginx 反代到后端 `8000`
- 或使用内网穿透工具（ngrok / Cloudflare Tunnel 等）得到一个临时 HTTPS 域名

随后把该 HTTPS 域名填到小程序端「我的 → 后端地址」。

## 4. 运行小程序（预览/扫码）

本项目已包含预览服务脚本，可生成二维码进行预览调试。

在项目根目录运行预览脚本后，复制输出的预览链接打开并按提示生成二维码即可。

### 4.1 生成二维码失败：invalid ip（-10008）

如果你在“生成微信二维码/预览”时遇到：

`errCode:-10008, errMsg: invalid ip: x.x.x.x`

这是微信的 CI 安全限制：调用 `miniprogram-ci` 的预览/上传接口时，**执行构建的机器公网 IP 必须在小程序后台的 IP 白名单内**。

解决方法：

- 登录微信公众平台（小程序）后台 → 开发 → 开发管理/开发设置（不同版本入口名称略有差异）→ 找到 CI/IP 白名单
- 把报错里的公网 IP（例如 `180.184.96.69`）加入白名单并保存
- 等待生效后，重新点击“生成二维码/预览”

注意：

- 如果你用的是云端构建环境，公网 IP 可能会变化，需要按报错提示更新白名单。
- 如果你不方便维护 IP 白名单，测试阶段可以改用本地微信开发者工具进行预览（不用走 CI 预览链路）。

## 5. 功能验证清单（本次实现）

### 5.1 周期性账单

1. Tab「我的」→ 周期性账单 → 新建模板
2. 选择支出/收入、金额、账户、分类、周期、下次生成日期，保存
3. 回到“周期性账单”页或“首页”自动触发拉取待确认列表
4. 进入“待确认账单”页 → 确认入账
5. 回到首页查看“最近账单”是否出现一笔新记录

### 5.2 导出 CSV

1. Tab「我的」→ 导出 CSV
2. 选择日期范围 → 开始导出
3. 小程序会下载并保存 CSV 文件，支持直接打开（不同平台对 CSV 的打开能力不同）
